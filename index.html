Server统计每张数据表的大小

发表于 2015-09-19

在 SQL Server 中，了解每个表的记录数、占用磁盘空间大小是非常有必要的。
我们可以使用 sp_spaceused 这个存储过程来查询表的信息。

sp_spaceused的使用方法如下：

  1. exec sp_spaceused ‘表名’ （SQL统计数据，大量事务操作后可能不准）
  2. exec sp_spaceused ‘表名’, true (更新表的空间大小，准确的表空大小，但可能会花些统计时间）
  3. exec sp_spaceused (数据库大小查询）
  4. exec sp_MSforeachtable “exec sp_spaceused ‘?’” (所有用户表空间表小，SQL统计数据，，大量事务操作后可能不准）
  5. exec sp_MSforeachtable “exec sp_spaceused ‘?’,true” (所有用户表空间表小，大数据库慎用）

下面这条 SQL 语句就可以查询每一个表的信息了：

CREATE TABLE #t(
name VARCHAR(255),
ROWS BIGINT,
reserved VARCHAR(20),
DATA VARCHAR(20),
index_size VARCHAR(20),
unused VARCHAR(20)
)
EXEC sp_MSforeachtable “insert into #t exec sp_spaceused ‘?’”
SELECT * FROM #t ORDER BY name
DROP TABLE #t

解决ionic的tabs在Android系统中一直处于顶部位置问题

发表于 2015-09-01

Ionic build APP之后，Android真机上测试tabs导航始终处于顶部位置

修改测试正常后的代码如下所示：

//解决tab导航位置Android始终在顶部，IOS始终在底部的问题
$ionicConfigProvider.platform.ios.tabs.style(‘standard’);
$ionicConfigProvider.platform.ios.tabs.position(‘bottom’);
$ionicConfigProvider.platform.android.tabs.style(‘standard’);
$ionicConfigProvider.platform.android.tabs.position(‘bottom’);
$ionicConfigProvider.platform.ios.navBar.alignTitle(‘center’);
$ionicConfigProvider.platform.android.navBar.alignTitle(‘center’);
$ionicConfigProvider.platform.ios.backButton.previousTitleText(‘’).icon(‘ion-ios-arrow-thin-left’);
$ionicConfigProvider.platform.android.backButton.previousTitleText(‘’).icon(‘ion-android-arrow-back’);
$ionicConfigProvider.platform.ios.views.transition(‘ios’);
$ionicConfigProvider.platform.android.views.transition(‘android’);

hexo服务启动正常浏览器却无法访问

发表于 2015-08-27

近日基于Coding+hexo搭建了博客，今天在写完一篇文章后在本地(Windows10)用命令启动hexo服务预览下：

$ hexo s

ssh显示服务已经成功启动在http://localhost/的4000端口，但是浏览器中的http://localhost:4000 url却无法访问

于是用ping命令ping http://localhost

ping http://localhost

得到的是可以ping通localhost, 再次访问了本地的IIS服务器（挂了一个在维护的比较老的ASP网站）, 该网站是绑定的8088端口，结果是可以正常访问的，所以推断问题应该没有出在localhost上

于是用netstat命令查看4000端口是否被占用：

可以看出有6条记录显示 127.0.0.1即localhost4000端口的状态是 CLOSE_WAIT， CLOSE_WAIT的状态显然是异常的，该进程的PID为1568

接下来直接使用了 tasklist 命令查找对应PID的进程名：

tasklist |findstr 1568

得到结果：

FoxitProtect.exe 1568 Services 0 14,904 K

I got you! 应该就是这货占用了原本属于hexo服务的端口，这个是福昕阅读器的一个进程，早前因为要看PDF重新装了这货

找到“罪魁祸首”就好办了，直接kill掉就行了

taskkill /T /F /PID 1568 强制干掉PID是1568的进程

重启hexo服务：

$ hexo s

发表于 2015-08-26

标题


标题是每篇文章都需要也是最常用的格式，在 Markdown 中，如果一段文字被定义为标题，只要在这段文字前加 # 号即可。

二级标题

三级标题

以此类推，总共六级标题，建议在井号后加一个空格，这是最标准的 Markdown 语法。

列表


熟悉 HTML 的同学肯定知道有序列表与无序列表的区别，在 Markdown 下，列表的显示只需要在文字前加上 - 或 * 即可变为无序列表，有序列表则直接在文字前加1. 2. 3. 符号要和文字之间加上一个字符的空格。

引用

如果你需要引用一小段别处的句子，那么就要用引用的格式。

只需要在文本前加入 > 这种尖括号（大于号）即可

图片与链接

插入链接与插入图片的语法很像，区别在一个 !号

图片为：![](){ImgCap}{/ImgCap}

链接为：[]()

插入图片的地址需要图床，这里推荐围脖图床修复计划 与 CloudApp 的服务，生成URL地址即可。

粗体与斜体

Markdown 的粗体和斜体也非常简单，用两个 包含一段文本就是粗体的语法，用一个 包含一段文本就是斜体的语法。
